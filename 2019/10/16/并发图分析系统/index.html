并发图分析系统
发布于：2019-10-16 09:22:33 标签：/ graph processing system / concurrent graph processing / paper reading / 访问：

并发图分析任务：大量的图算法并发的运行在同一平台上对底层同一个图数据进行处理，以对该图数据进行多方位的分析处理，获得各种目的性的分析结果。

研究工作

Yu Zhang’s Research

CGraph

基于关联性感知的并发图处理

  1. CGraph: A Correlations-aware Approach for Efficient Concurrent Iterative Graph Processing(ATC’18)
  2. CGraph: A Distributed Storage and Processing System for Concurrent Iterative Graph Analysis Jobs(TOS’19)

并发图分析任务之间存在关联性。

（1）空间关联性：多个并发图分析任务需要访问和处理的图数据存在大量的交集。

（2）时间关联性:多个并发图分析任务可能需要同时访问同一个图划分块。

但在此前的图处理系统中进行并发图分析任务时，各个任务对共享图数据的访问相互独立，互不感知。使得各个并发的图分析任务在计算时遇到内存墙（内存性能严重限制CPU）和缓存相互干扰的问题。

这篇工作通过感知各个并发图分析任务之间的关联性，使得各个并发图分析任务之间能有效的共享数据和数据访问。

DiGraph

基于关联依赖的单任务图处理

  • DiGraph: An Efficient Path-based Iterative Directed Graph Processing System on Multiple GPUs(ASPLOS’19)

  • Efficient Disk-Based Directed Graph Processing: A Strongly Connected Component Approach(TPDS 2018)

同一个图分析任务的子任务（处理各图划分块的任务）之间也存在依赖关联性。
核心图顶点状态收敛所需要的更新次数决定了整个图收敛所需迭代次数。

这篇工作在单个图分析任务场景下，通过感知图中顶点之间的依赖关联性特征进行图划分，使得各个图划分块之前的依赖关联性尽量符合全序关系，尝试使得未收敛的图划分块对已经收敛的图划分块影响最小化。另外通过加速核心顶点之间的状态传递，进一步加速全局的收敛速度。

FBSGraph
